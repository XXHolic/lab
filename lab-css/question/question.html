1 标准的 CSS 的盒子模型与低版本 IE 的盒子模型有什么不同的？

标准的盒子模型：宽度 = width + border + padding + margin

低版本 IE 盒子模型：宽度 = width(包含了 border + padding) + margin

2 box-sizing 属性作用？

  定义了该如何计算一个元素的总高度和总宽度。
  默认是 content-box，这时属性 width/height 的值就是内容的宽高度，
  当值为 border-box 时，width/height 属性包括了内容、内边距和边框。

3 CSS 选择器有哪些？哪些属性可以继承？

  CSS 选择器有：
  id 选择器
  类选择器
  标签选择器
  后代选择器
  相邻选择器
  子选择器
  属性选择器
  伪类选择器
  可继承属性：
  font、line-height、text-align、color、direction、visibility、cursor 等等。
  不可继承属性：
  display、width、height、margin、padding、border、background、float、position 等等。

7 常见的兼容性问题

  1 不同浏览器 margin 和 padding 默认值不一样
  2 chrome 下字体小于一定值会无效，-webkit-text-size-adjust: none;
  3 连接点击后 hover 样式不出现，解决方法是书写 css 顺序：a:link {} a:visited {} a:hover {} a:active {}

8 absolute 的 containing block 计算方式跟正常流有什么不同

static/relative 的包含块是最近块级父元素的内容框
absolute 的包含块是由最近的position非static的祖先元素：
  如果是块级元素，包含块是为该元素的内边距边界；
  如果祖先是行内元素，那么包含块是第一个和最后一个为该元素生成的框的包围盒；
  如果没有祖先元素，元素的包含块就是初始包含块（由用户代理建立的，视窗大小的矩形区域）;
fixed 的包含块是视窗。

9 CSS 里的 visibility 属性有个 collapse 属性值？在不同浏览器下以后什么区别

collapse 这个值用在除了表格元素之外的元素，呈现跟 hidden 一样，用在 table 上时，会删除一行或一列，
但是它不会影响表格的布局。被行或列占据的空间会留给其他内容使用。

用在表格元素上时：
  chrome 上表现的 hidden 一样，不符合描述。现在已经修复了。
  火狐 ie 上 table 的行消失了，符合描述。
